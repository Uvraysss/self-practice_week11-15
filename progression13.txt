
สรุปสิ่งที่ได้เรียนรู้ในสัปดาห์ที่ 13
    
    ในสัปดาห์นี้จากการทำ self-practice ได้เข้าใจภาพรวมของการพัฒนาเว็บฝั่ง Client มากขึ้น โดยเฉพาะการทำงานของ DOM การจัดการเหตุการณ์ 
    (Event Handling) การควบคุม logic ของโปรแกรม และการจัดการวันเวลา ซึ่งเป็นพื้นฐานสำคัญ เริ่มตั้งแต่การฝึกสร้าง To-Do List แบบไดนามิก 
    ได้เรียนรู้ว่าหน้าเว็บสามารถเปลี่ยนแปลงได้ ตลอดเวลาจาก JavaScript โดยการเข้าถึงองค์ประกอบใน DOM เช่น input, ปุ่ม และรายการ <ul> 
    ผ่านฟังก์ชัน document.getElementById() หรือ document.querySelector(). จากนั้นได้ฝึกการสร้าง element ใหม่แบบ runtime เช่น <li> 
    และ <button> รวมถึงการใส่ Event Listener ให้ element ที่เพิ่งสร้างขึ้นแบบไดนามิก เช่นปุ่ม Delete ที่ลบตัวเองออกจากรายการ ซึ่งช่วยให้เข้าใจ
    โครงสร้างของ DOM และการจัดการอีเวนต์ได้ดียิ่งขึ้น และได้ฝึกทำระบบกรองข้อมูลในตารางแบบ real-time ผ่านช่องค้นหา ซึ่งช่วยทำให้เข้าใจว่า event 
    'input' ทำงานอย่างไร เมื่อผู้ใช้พิมพ์แต่ละตัวอักษร JavaScript จะตรวจสอบค่าใหม่ทันทีและนำไปเช็คกับข้อความในแถวของตารางผ่าน 
    row.textContent.includes() แล้วซ่อนหรือแสดงแถวตามผลลัพธ์ สิ่งนี้ช่วยพัฒนาทักษะการวนลูปและคิดลอจิกเกี่ยวกับการกรองข้อมูล ซึ่งเป็นโครงสร้าง
    พื้นฐานที่เว็บหลาย ๆ เว็บไซต์ใช้อยู่จริง ได้เรียนรู้การทำงานเกี่ยวกับเวลาโดยใช้ setInterval() ในการสร้างตัวนับถอยหลังแบบง่าย การตั้งเวลาให้โค้ดทำ
    งานซ้ำทุกหนึ่งวินาที และการลดค่าตัวแปรลงทีละ 1 ทำให้เข้าใจการจัดการ interval และการหยุดมันด้วย clearInterval() ซึ่งเป็นแนวคิดสำคัญสำหรับ
    การสร้างฟังก์ชันที่ต้องทำงานตามเวลา เช่น timer, animation หรือระบบแจ้งเตือน ในส่วนของการจัดการวันเวลา ยังได้ฝึกดึงวันที่และเวลาปัจจุบันโดยใช้ 
    new Date() และนำข้อมูลที่ได้มา formatting ให้เป็นรูปแบบ “YYYY-MM-DD HH:MM:SS” ซึ่งเกี่ยวข้องกับการใช้เมธอดเช่น getFullYear(), getMonth(), 
    getDate(), getHours() และการแก้ปัญหาค่าเดือนที่เริ่มนับจาก 0 รวมถึงการใช้ padStart() เพื่อจัดเลขให้เป็นสองหลักเสมอ เป็นสิ่งที่จำเป็นมากในระบบที่
    ต้องบันทึก log หรือแสดงเวลาที่มีมาตรฐาน สุดท้ายได้เรียนรู้การใช้ getDay() เพื่อตรวจสอบว่าวันปัจจุบันเป็นวันเสาร์หรือวันอาทิตย์หรือไม่ ซึ่งช่วยให้เข้าใจการ 
    mapping ค่าวันในรูปแบบตัวเลข 0–6 และการเขียนเงื่อนไขแบบ boolean return ที่กระชับ และชัดเจน
